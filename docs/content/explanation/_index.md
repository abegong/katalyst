# Explanation

Understanding-oriented discussion of the *why* behind Katalyst: the vision,
the general data-systems model it descends from, and how those ideas grow
into the shipped tool. These pages serve both users who want the bigger
picture and contributors who need the conceptual map.

Rationale that is tied to a specific subsystem lives next to the code, in the
per-package `README.md` files under `internal/` (for example
`internal/config`, `internal/frontmatter`, and `internal/checks`). The pages
here cover the cross-cutting *why* that no single package owns.
