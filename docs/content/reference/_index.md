# Reference

Information-oriented descriptions of the configuration surface, the rule
checks, and the project vocabulary. Reference pages describe *what is*, not
*how to* — they are looked up, not read front to back. Rule pages under
[Rules]({{< relref "rules/_index.md" >}}) are generated from the checks
registry, so they never drift from the code.
